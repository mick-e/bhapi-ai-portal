# mobile/apps/safety/native/

Source files for custom React Native native modules that the Safety app
depends on. Organised by platform:

- `ios/` — Swift + Objective-C bridges compiled into the iOS build
- `android/` — Kotlin bridges compiled into the Android build

## Why this directory exists

`mobile/.gitignore` ignores any `ios/` or `android/` directory — these are
generated by `expo prebuild` and never committed. But native module
**source** files must be checked in so every app build can pick them up.

These files are force-added via `git add -f` on commit. Do NOT remove
them or the native modules will disappear on the next prebuild.

## Registration

- **iOS:** `PermissionKitBridge.m` exports the Swift class to the
  React Native bridge. No further registration needed; React Native
  auto-discovers `RCT_EXTERN_MODULE`.
- **Android:** `DigitalWellbeingPackage` must be added to
  `MainApplication.kt`'s `getPackages()` list. The `expo prebuild`
  process regenerates `MainApplication.kt`, so use a config plugin
  (`plugins/with-digital-wellbeing.js`) to inject the import + add
  the package. TODO: plugin lands with the Task 30 hardening batch.

## Testing

Native modules can't be exercised in CI. See
`mobile/packages/shared-native/__tests__/integration.md` for the
manual test plan.
