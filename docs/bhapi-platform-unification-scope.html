1. Executive Summary

This document defines the scope, deliverables, timeline, and resource requirements for the Bhapi Platform Unification programme — a 6-month initiative to consolidate all Bhapi products into a single, market-competitive platform under the bhapi.ai domain.

The Challenge

Bhapi currently operates three separate codebases (AI Portal, Social App, Back-Office) with fragmented user experiences, zero test coverage on legacy systems, 18 unresolved security vulnerabilities, and three imminent regulatory deadlines. Competitors — GoGuardian, Bark, and Aura — are expanding aggressively into adjacent markets.

The Solution

Unify all products into a single Bhapi Platform repository with shared authentication, a consolidated database, and one deployment pipeline. Archive legacy systems. Build a greenfield mobile application. Ship competitive features to defend and expand market position.

Regulatory Deadlines — Non-Negotiable

COPPA 2026: April 22, 2026 — FTC enforcement, fines up to $50,120 per violation

Ohio AI Mandate: July 1, 2026 — State governance requirements for schools

EU AI Act: August 2, 2026 — High-risk AI system conformity requirements

3.1 Sub-Plan 1: COPPA 2026 Compliance

Deadline: April 22, 2026 Effort: 4–5 person-weeks Risk if missed: FTC fines up to $50,120 per violation

Context

The FTC's updated COPPA rule introduces five new requirements. The AI Portal has partial compliance (consent flows exist, COPPA dashboard exists, Yoti age verification integrated). Gaps remain in third-party consent itemisation, written data retention policy, VPC strengthening, "refuse partial collection" toggle, and push notification consent separation.

Existing Infrastructure

Migration 031_coppa_2026.py already creates tables: third_party_consent_items, retention_policies, push_notification_consents, video_verifications. New work builds on these existing tables.

Task 1.3: Refuse Partial Collection Toggle

Objective

COPPA 2026 grants parents the right to consent to data collection while refusing third-party sharing. Implement a group-level toggle that, when enabled, automatically revokes all non-integral third-party consents.

Specification

  • New column refuse_third_party_sharing (Boolean) on Group model
  • When toggled ON: all non-integral third-party consents are automatically revoked
  • Integral services (e.g., Stripe for billing) are exempt from bulk revocation
  • Accessible via PATCH /api/v1/groups/{id}/settings

Task 1.4: VPC Enhancement (Verifiable Parental Consent)

Objective

COPPA 2026 eliminates knowledge-based VPC methods. Implement SMS + confirmation as the primary VPC mechanism.

API Endpoints

MethodEndpointDescription
POST/api/v1/auth/vpc/initiateSend verification SMS to parent's phone
POST/api/v1/auth/vpc/verifyConfirm consent with verification code

Flow

1. Parent provides phone number → 2. System sends SMS with 6-digit code → 3. Parent enters code → 4. System records verified consent with timestamp and method
